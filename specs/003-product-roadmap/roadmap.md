# PPT_Agent 产品优化方向规划（M6–M8）

> 承接 [002-ppt-master-comparison/analysis.md](file:///f:/workspace/PPT_Agent/specs/002-ppt-master-comparison/analysis.md) 的对标结论，将"差异化护城河"沉淀为可执行的产品路线图。本规划面向 **2026 H2** 的 3 个里程碑（M6/M7/M8），覆盖功能补齐、护城河深化、平台化与商业化前置。

---

## 1. 一句话定位（产品愿景）

> **PPT_Agent = 企业级"风格对齐"的 AI PPT 平台** — 上传自家样本，让 AI 学会"我司的 PPT 怎么写、怎么画、怎么讲"，在合规与协作框架内被全员复用。

### 与 PPT-master 的边界

| 我们**做**的 | 我们**不做**的 |
| --- | --- |
| 风格对齐、知识沉淀、协作审计 | 单机本地工作流 / Skill 内分发 |
| 多人多端、企业级权限 | 单 IDE 内对话驱动 |
| 量化质量与可观测 | "一把出完美"一次性体验 |

## 2. North Star Metric & 子指标

**北极星指标**：**周活用户生成的、可被再次编辑的 PPT 页数**（"可被再次编辑" = 通过样式评分 ≥ 0.7 且导出后用户在 PowerPoint 中至少二次打开）

| 维度 | 指标 | M6 目标 | M7 目标 | M8 目标 |
| --- | --- | --- | --- | --- |
| **采纳** | DAU / MAU | 30% | 40% | 50% |
| **质量** | 平均样式评分（overall） | 0.72 | 0.78 | 0.82 |
| **效率** | 端到端生成时延 P50 | 4 min | 3 min | 2 min |
| **效率** | 阶段 ReDo 节省率 | ≥ 50% | ≥ 65% | ≥ 75% |
| **留存** | 次周留存 | 35% | 45% | 55% |
| **护城河** | 知识库中沉淀样本数 / 用户 | 8 | 20 | 50 |
| **商业前置** | 私有化部署客户数 | 1 (Pilot) | 3 | 8 |

## 3. 五大产品优化主题

### 主题 A：**风格对齐（Style Alignment）再升级** — 核心护城河

把"上传样本 → 风格一致生成"做成**业界第一**。

- **A1 模板抽取器**（P1）：用户上传 `.pptx` 模板 → 自动抽取主色/字体/母版/页脚/Logo → 存为 `TemplateStyle` → 后续生成可"沿用"该模板。
- **A2 风格 → 视觉规范编译器**：把样本的 layout/palette/font 评分结果 + 模板抽取结果**编译**为 LLM 友好的 `VisualStyleSpec`，避免每页重复 prompt。
- **A3 跨页一致性保险**：在 SVG 阶段加入"风格约束检查器"，越界自动修正（如标题字号抖动 > 10% 即触发重排）。
- **A4 风格 A/B 工作台**：UI 上提供"原始风格 vs. 优化风格"并排对比 + 评分曲线，沉淀为回归数据集。

> 验收：A1 上线后"模板沿用"任务的成功率 ≥ 85%（用户接受率）。

### 主题 B：**生成链路能力补齐** — 缩短与 PPT-master 的能力差距

- **B1 动画 & 转场**（P0）：默认开启淡入/擦除，飞入作为可选；UX 端提供"是否启用动画"开关；引入 `animation_preset` 字段。
- **B2 演讲者备注 → 旁白 TTS**（P0）：备注已存在；接入 DashScope / OpenAI / Edge TTS，输出 MP3 嵌入 PPTX 备注页；提供"品牌声音"占位。
- **B3 多画幅**（P1）：后端抽象 `CanvasSpec`（16:9 / 4:3 / 3:4 / 1:1 / 9:16 / A4），DB 落字段；前端 Preview 自适应。
- **B4 AI 配图**（P1）：在 SVG 之后、PPTX 之前插入 `image_resolver`：优先复用 Material 库 → 回退到 Pexels/Pixabay → 最后 gpt-image-2；图片合规/PII 走现有中间件。
- **B5 更多输入格式**（P2）：PDF / DOCX / HTML / EPUB / URL 通过 Unstructured / MarkItDown 统一转 MD 后进现有 chunk → embed 链路。
- **B6 端到端流式预览**：WebSocket 已具备，叠加前端"边生成边渲染"骨架屏（先生成 SVG → 立即前端预览 → 后台异步转 PPTX）。

> 验收：B1+B2+B3 上线后，"用户接受率"提升 ≥ 20%；B4 配图缺图率从 ~40% 降到 ≤ 10%。

### 主题 C：**协作与编辑体验** — 把"生成"升级为"协作"

- **C1 草稿多人实时协作**（P1）：基于现有 Lock 机制扩展为多人光标 / 评论 / @提及。
- **C2 Outline 编辑器**：在 outline 阶段允许用户直接拖拽排序 / 改标题 / 删页，前端所见即所得；后端 `outline` stage 支持 partial update。
- **C3 选区级 Redo**：当前 stage 级 ReDo，升级为"选区级"（只重生成某一页的某一处）。
- **C4 模板市场**：内置 + UGC 模板广场，按行业 / 风格 / 沟通模式筛选。
- **C5 版本与回滚**：Draft 支持"快照"，可在历史版本间穿梭对比。

> 验收：C1 上线后"团队周活"占比 ≥ 30%。

### 主题 D：**知识库与代理进化** — 长期复利

- **D1 知识库健康度面板**：覆盖率、命中率、过期样本数；自动建议"补料"。
- **D2 偏好自学习 v2**：从"被采纳/被否决"的页迭代偏好权重；形成个人/团队双层偏好。
- **D3 跨任务素材复用**：把"上次生成的好页"自动归档为下次可调用的 Reference Slide。
- **D4 Agent 自评估**：在每个 stage 后跑 LLM-as-Judge 自评，不达标自动重做；纳入 Trace。
- **D5 反馈闭环**：UI 一键 "👍 / 👎 / 改这里" → 进入训练集 / 偏好微调。

> 验收：D4 自评估后"用户无修改通过率"提升 ≥ 15%。

### 主题 E：**平台化与商业化前置** — 从工具到产品

- **E1 计费与配额**：按页 / 按 token / 按存储分层计费；与现有 `Credential` 体系对齐。
- **E2 多租户 + 团队空间**：团队共享知识库 / 模板 / 偏好；RBAC（Owner/Admin/Editor/Viewer）。
- **E3 私有化部署**（P1→P2）：Helm Chart + 离线模型仓库 + 内网对象存储适配。
- **E4 开放 API + Webhook**：让外部系统（CRM、OKR、BI）触发"自动生成汇报 PPT"。
- **E5 审计与合规导出**：SOX / GDPR / 等保 2.0 三级对照检查清单；审计日志可导出。

> 验收：M6 完成 E1+E2+MVP 计费；M7 完成 E3 私有化 PoC；M8 完成 E4+E5。

## 5. 非目标（Non-Goals）

为了聚焦，下列事项**本期不做**：

- 自研 PPTX 解析器（继续用 python-pptx + 自研 SVG→DrawingML）。
- 自研 Embedding 模型（用 OpenAI / BGE / DashScope 即可）。
- 取代 PowerPoint 的"在线编辑器"（我们是"生成 + 导出"，不是"在线 Office"）。
- 移动端原生 App（先用响应式 Web 满足）。
- 通用聊天机器人 / 写作助手（保持"演示文稿"垂直）。

## 6. 风险与缓解

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| LLM 生成 SVG 不稳定导致接受率低 | 高 | A3 一致性保险 + D4 自评估 + 失败回退到 _fallback_svg |
| TTS / AI 配图供应商成本失控 | 中 | 多通道 + 缓存 + 用户级开关 |
| 私有化部署复杂度过高 | 中 | M8 只做 PoC，不承诺 SLA；M9+ 再深入 |
| 模板抽取器对客户私域模板的版权问题 | 中-高 | 模板市场 UGC 必须确认版权；内置模板走原创/CC 协议 |
| 协作功能引入复杂度爆炸 | 中 | 先做 C1 评论 + @，光标同步留到 M9+ |
| 模型依赖强（OOS 模型下质量骤降） | 中 | D2 偏好自学习 + 评分体系作为"质量底盘" |

## 7. 跨里程碑数据飞轮

```
      ┌──────────────┐
      │  用户上传样本  │
      └──────┬───────┘
             ▼
      ┌──────────────┐    ┌────────────────┐
      │  知识库 + 偏好 │◀──│  接受 / 拒绝反馈 │
      └──────┬───────┘    └────────┬───────┘
             ▼                      ▲
      ┌──────────────┐              │
      │  风格对齐生成  │──────────────┘
      └──────┬───────┘
             ▼
      ┌──────────────┐
      │  评分 + Trace │
      └──────┬───────┘
             ▼
      ┌──────────────┐
      │  回归 / 自评估 │
      └──────────────┘
```

每一次"接受"与"拒绝"都回流到知识库与偏好，长期形成**对手短期无法复制的数据护城河**。

## 8. 总结

- **M6**：补齐 PPT-master 已有能力（动画、旁白、多画幅、流式预览），让生成结果"立刻可用"。
- **M7**：深化护城河（模板抽取 + 视觉规范 + 跨页一致性），开启协作（团队空间 + Outline 编辑）。
- **M8**：平台化（计费、API、私有化、合规），把工具沉淀为产品。

战略上以 **"风格对齐 + 数据飞轮"** 为长期护城河，以 **"能力补齐 + 协作 + 平台化"** 为 12 周主轴。最终目标：让 PPT_Agent 成为**"企业做自家 PPT"的默认入口**。
