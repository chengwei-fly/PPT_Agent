# Feature Specification: PPTagent MVP 业务闭环（生成 / 知识库 / Agent进化三大主线）

**Feature Branch**: `001-mvp-closed-loop`
**Created**: 2026-06-24
**Status**: Draft
**Input**: 面向产品与方案人员的垂类智能 PPT 生成与管理平台 MVP 闭环，覆盖 12 周路线图 M1–M3 三大主线（生成 / 知识库 / Agent 进化）。

## Clarifications

### Session 2026-06-24

- Q: PII 检测中间件命中后的处置策略（整文件拒绝 / 字段级处置 / 整文件放行） → A: 字段级处置 — 保留文档，对命中字段按规则（替换为占位符 / 脱敏星号）处理，并在解析摘要中标注；同时落安全事件记录。
- Q: MVP 阶段可用性 SLA 目标 → A: 月度可用率 ≥ 99%，核心故障（生成/解析/偏好提取链路完全不可用）≤ 4 小时恢复，按工作日响应；维护窗口提前 24 小时公告。
- Q: 样本 / 偏好 / 生成任务默认保留时长 → A: 样本与偏好规则长期保留（用户不主动删则不自动过期）；生成任务结果保留 180 天，到期前 14 天向用户发出"续期 / 归档 / 导出"三选一通知，逾期未操作则自动归档至冷存储且对前端不可见，原始下载链接失效。
- Q: 风格契合度的客观评分口径 → A: 客观指标自动化打分为主（版式结构 / 配色分布 / 字体族三类硬指标，每类命中 ≥ 80% 视为"风格契合"），辅以每月 1 次对 5% 抽样做 3 名产品经理盲评平均 ≥ 4 分（5 分制）作为"主观体验"补丁。
- Q: 同一用户并行生成任务数与队列行为 → A: 单用户最多 2 个并发生成任务；第 3 个起入队等待，UI 顶部以常驻条展示"当前排队位置 / 预计等待时长"，并提供"取消旧任务让位"动作；排队超过 5 分钟未执行则自动取消并通知用户。

## 术语定义

- **风格 (Style)**：指 PPT 的视觉表现体系，由三个可度量子维度组成：① 版式结构（页面布局、元素位置、留白比例）② 配色分布（主色/辅色/强调色的色值及面积占比）③ 字体族（标题/正文/标注各层级的字体、字号、字重组合）。"风格一致"或"风格契合"的判定标准见 SC-002（客观三层命中均 ≥80%）。
- **同类型修改 (Same-type Edit)**：指在生成轨迹的同一阶段（大纲/要点/SVG 排版/导出）、对同一类页面元素（封面/正文/数据页/结语）、执行相同方向的调整（如"封面 logo 移至右上角"），且两次修改间隔不超过 30 天。
- **草稿整体风格 (Draft Overall Style)**：指草稿中占多数（按页数 ≥60%）的页面所呈现的风格聚类结果，由系统自动计算并缓存；当用户手动覆盖某页样式后，整体风格在下次保存时重新计算。

## User Scenarios & Testing *(mandatory)*

PPTagent Constitution v1.0.0 要求本 spec 覆盖知识库 / 生成 / Agent 进化三条主线，本 spec 在 P1+P2 阶段完整覆盖三大主线，P3 补足安全与可控性。下文按用户旅程优先级组织，每条用户故事均可独立开发、测试、交付。

### User Story 1 - 一句话生成风格对齐的 PPT (Priority: P1) 🎯 MVP 核心
作为产品 / 方案人员，我用一句话描述需求（例如"做一份 Q3 新能源储能项目立项汇报，15 页"），系统根据我既有的样本 PPT 风格，输出一份可直接打开的 PPTX 草案，并允许我在线逐页调整。

**Why this priority**：
- 这是项目 MVP 闭环的核心入口，验证"个人知识库 + AI 生成"是否真正替代人工制作。
- 决定 12 周路线图 M1 的可交付物是否成立，是后续所有 P2/P3 故事的依赖。

**Independent Test**：
- 准备 3 份不同风格的样本 PPT 进入知识库。
- 在空知识库与已存在 3 份样本两种状态下分别触发"一句话生成"，对比生成结果的版式契合度与可读性。
- 生成流程结束后，无需任何后端持久化操作也能在浏览器直接下载 PPTX 文件。

**Acceptance Scenarios**：
1. **Given** 用户已登录且知识库中至少存在 1 份样本 PPT，**When** 用户输入"做一份 12 页的项目立项汇报，重点突出市场规模和投资回报"，**Then** 系统在 5 分钟内输出一份不少于 10 页的 PPTX 文件，且首页与正文至少 80% 的版式与样本 PPT 同类页面风格一致（按"术语定义"中风格三维度客观判定）；剩余不一致的页面在风格契合度报告中标注具体差异维度，用户可选择"接受"或"重做该页"。
2. **Given** 用户刚登录且知识库为空，**When** 用户输入任意一句话，**Then** 系统给出"知识库为空，请先上传样本"的明确提示并引导上传，不允许直接生成无风格依据的 PPT。
3. **Given** 生成正在进行中，**When** 用户点击"取消"，**Then** 系统在 5 秒内停止 LLM 调用与渲染，释放占用的算力配额，并在历史记录中留下"已取消"状态。
4. **Given** 同一用户已有 2 个生成任务正在执行，**When** 用户发起第 3 个生成任务，**Then** UI 顶部出现排队条显示排队位置与预计等待时长；5 分钟内未开始则任务自动取消并通知用户。

### User Story 2 - 上传并管理个人 PPT/PDF/Word 样本 (Priority: P1) 🎯 MVP 核心
作为产品 / 方案人员，我把日常积累的 PPT/PDF/Word 资产批量上传到个人知识库，系统解析后供后续生成调用。我可以查看已上传样本、删除样本、查看样本对哪些生成任务产生了影响。

**Why this priority**：
- "个人知识库"是 PPTagent 与通用 AI PPT 工具的核心差异点（见产品规划报告 §1.1）。
- 没有可用样本库，US1 的"风格对齐"承诺就是空头支票。
- 用户的样本文件是高敏感数据，存储与解析策略直接决定 Constitution §IV（数据安全前置）是否被满足。

**Independent Test**：
- 上传 5 份不同版式（汇报 / 培训 / 方案 / 数据 / 营销）的 PPT 后，列表正确显示 5 条记录。
- 解析失败的样本在列表中明确标注"解析失败 + 失败原因"，不阻塞其他样本。
- 删除某一样本后，依赖该样本生成的旧 PPT 仍可继续下载（解耦存储）。

**Acceptance Scenarios**：
1. **Given** 用户打开知识库页面，**When** 用户拖入 5 份 PPTX 文件，**Then** 系统在 2 分钟内完成全部 5 份的解析，列表显示文件名 / 解析状态 / 解析时间 / 解析页数。
2. **Given** 用户上传的样本中含手机号 / 客户名等可识别信息，**When** 解析完成，**Then** 系统在摘要中自动去除这些 PII 字段，且原始文件按"原始 / 解析 / 向量"三类分离存储。
3. **Given** 用户上传 1 份损坏的 PPTX，**When** 解析失败，**Then** 列表中明确显示"解析失败"与原因摘要，允许用户重新上传或删除该条目，不影响其他样本。

### User Story 3 - Agent 学习并应用我的偏好 (Priority: P2)
作为产品 / 方案人员，我希望 Agent 在多次生成 / 修改后，能识别我反复强调的偏好（例如"封面一定要放公司 logo"、"每页不超过 5 行要点"），并在新任务中主动应用，而不需要我每次重复说明。

**Why this priority**：
- 这是 Constitution §III（可解释可控制）与 §V（可观测可追溯）的具体落地，决定 Agent 进化引擎是否真的"越用越好用"。
- 对应 12 周路线图 M3 交付物。
- 仍属"增强体验"层，缺失不会让 MVP 失败，但缺失会让差异化优势无法兑现。

**Independent Test**：
- 在 5 次生成任务中，每次都对封面样式执行相同修改（如"把 logo 放到右上角"）。
- 第 6 次生成新任务时，系统应自动应用同样的封面样式，并附"依据：你之前 5 次在封面做了相同调整"的解释。
- 用户对单次自动应用结果"撤销 / 锁定原文"时，Agent 不应再次自动应用该偏好。

**Acceptance Scenarios**：
1. **Given** 用户在历史生成中对封面样式做过至少 3 次相同调整，**When** 用户开启新任务且不显式描述封面要求，**Then** Agent 在生成轨迹中输出"已应用偏好 P-007（封面 logo 右上角），依据：你过去 3 次类似调整"且最终 PPT 实际应用了该样式。
2. **Given** Agent 已学习到某条偏好，**When** 用户在某次生成中点击"撤销 / 锁定原文"覆盖该偏好，**Then** 该次生成不应用该偏好，且 Agent 在该用户维度上把此偏好标记为"已忽略一次"，但不删除规则。
3. **Given** 用户希望清理某条偏好，**When** 用户在"我的偏好"页面查看并删除，**Then** 该偏好立即从新任务的候选集中移除，旧生成结果不受影响。

### User Story 4 - 生成轨迹可回看与可回滚 (Priority: P2)
作为产品 / 方案人员，我希望看到一份 PPT 究竟是怎么生成的：哪一步是 LLM 写的要点、哪一步是系统自动套的版式、哪一步参考了哪些样本。我可以基于这份轨迹单独修改某一步，而不必重头再来。

**Why this priority**：
- 直接服务于 Constitution §III（可解释可控制）与 §V（可观测可追溯）。
- 是企业用户（方案人员 / 咨询顾问）"敢不敢在客户面前打开用"的关键信任锚点。
- 缺失会让 P1 故事停留在"AI 工具很好玩"层面，无法转化为付费订阅。

**Independent Test**：
- 完成一次生成后，用户能在"生成轨迹"页看到至少 4 个阶段：① 大纲生成 ② 单页要点 ③ SVG 排版 ④ PPTX 导出，每阶段含输入摘要 / 输出摘要 / 耗时 / 引用的样本 ID。
- 点击任意阶段，可"重做此阶段"并把后续阶段标记为"待重新生成"。

**Acceptance Scenarios**：
1. **Given** 用户刚完成一次生成，**When** 用户点击"查看生成轨迹"，**Then** 系统按时间顺序展示至少 4 个阶段的卡片，每张卡片包含：阶段名 / 输入摘要 / 输出摘要 / 耗时 / 引用的样本 ID。
2. **Given** 用户对"第 3 阶段 SVG 排版"结果不满意，**When** 用户点击"重做第 3 阶段"，**Then** 系统仅重新执行第 3 阶段（含 SVG 排版与第 4 阶段导出），前 2 阶段结果保持不变，且耗时应明显小于完整重做。
3. **Given** 任意阶段执行失败，**When** 用户查看轨迹，**Then** 失败阶段以红色明确标识，并提供"重试 / 跳过 / 反馈"三个动作。

### User Story 5 - 数据安全与一键删除 (Priority: P3)
作为企业用户，我需要看到一份清晰的数据安全承诺，并能一键导出 / 删除我在系统中保存的全部数据，以便在公司合规审计时快速配合。

**Why this priority**：
- 直接对应 Constitution §IV（数据安全前置）与 §合规与安全基线。
- 对个人 / 团队用户影响小，但对"团队版 / 企业版"订阅决策具有一票否决权。
- 属于"必须做但优先级在核心闭环之后"的项。

**Independent Test**：
- 在"账户与数据"页一键导出 ZIP 包，ZIP 内含用户上传的原始文件、解析后的结构化摘要、用户偏好规则的可读 JSON，不含 LLM 提供商的内部记录。
- 一键删除后，用户所有样本 / 偏好 / 生成记录在 24 小时内从生产库与向量库中不可恢复地清除。

**Acceptance Scenarios**：
1. **Given** 用户至少有 1 份样本与 1 条历史生成记录，**When** 用户点击"导出我的全部数据"，**Then** 系统在 10 分钟内生成一个可下载的 ZIP 包，文件名包含"PPTagent-Export-{user_id}-{date}"。
2. **Given** 用户点击"删除我的全部数据"并二次确认，**When** 删除执行完成，**Then** 用户的样本库 / 偏好 / 生成记录在生产库中不再可查询，且在 7 天后从所有备份中清除。
3. **Given** 用户在 LLM 调用中触发了 PII 检测中间件告警，**When** 用户查看"安全事件"列表，**Then** 列表显示"何时调用 / 命中字段 / 处置方式（替换 / 拒绝 / 放行）"。

### Edge Cases

- **超长文档**：用户上传 100MB+ 的 PPT/PDF 时，系统应给出明确上限提示并拒绝超限文件，不允许触发 OOM。
- **空知识库下的"风格迁移"**：用户在新设备登录但本地知识库为空时，系统应明确"未匹配到个人风格，将使用通用模板"而非静默给出默认结果。
- **LLM 配额耗尽**：当用户本月 Token 配额用尽时，生成任务在排队前给出"配额已用完，下月 1 日重置"提示，已排队任务被取消且不消耗额外配额。
- **网络中断**：用户在生成过程中断网，30 分钟内重连可恢复任务状态（已完成的阶段结果保留），超过 30 分钟视为任务失败并清理中间产物。
- **样本重复**：用户重复上传同一份 PPT（哈希相同），系统应自动跳过解析并提示"该样本已存在"，不重复占用存储。
- **孤儿素材清理**：当样本被删除后其派生的 SlideAsset 成为孤儿素材时，系统应在"我的素材"页面以"来源已删除"标签醒目标识，并在孤儿素材数量 ≥10 时在页面顶部展示批量清理入口（"一键清理所有孤儿素材"）；单条孤儿素材仍支持用户逐条决定保留或删除。
- **跨语言样本**：用户上传英文 / 日文样本时，系统应按"用户当前界面语言"提示是否切换处理策略（默认保留原始语言，不强制翻译）。
- **草稿为空导出**：用户在草稿中无任何页面时点击"导出"，系统应拒绝导出并提示"草稿为空，请先添加页面"，不生成空 PPTX 文件。
- **锁到期未保存**：用户编辑草稿超过 30 分钟无操作导致写者锁到期时，系统应自动保存当前变更到服务端（以最后自动保存版本为准），并在锁释放后 5 分钟内向用户发送通知"您的草稿编辑权已释放，变更已自动保存"；若用户在锁到期前有未触发自动保存的变更（距上次保存 <5 秒），以最后一次自动保存结果为准。
- **删除与生成竞态**：用户发起"删除全部数据"后、清除执行完成前，若有正在运行的生成任务，系统应先取消该任务（FR-003 逻辑）再执行清除；新发起的生成请求在清除期间应返回 409 状态码并提示"数据清除进行中，请稍后再试"。
- **多设备并发登录**：同一用户在多个设备同时登录时，生成任务配额（FR-029 的 2 个并发限制）按 user_id 全局共享，不按设备/会话独立计算。

## Requirements *(mandatory)*

### Functional Requirements

**生成主线 (P1)**

- **FR-001**：系统 MUST 接受用户以自然语言输入的"一句话需求"，并产出一份不少于 10 页的 PPTX 文件作为最小可交付物。
- **FR-002**：系统 MUST 在知识库为空时拒绝直接生成并引导用户先上传样本。
- **FR-003**：系统 MUST 暴露"取消生成"入口，在用户触发后 5 秒内停止后续 LLM 调用与渲染任务。
- **FR-004**：系统 MUST 在生成前预估并展示预计 Token 消耗与预计耗时（基于历史同类任务的中位数）。
- **FR-005**：系统 MUST 支持生成结束后一键下载 PPTX 文件，文件名包含生成时间戳与任务 ID。

**知识库主线 (P1)**

- **FR-006**：系统 MUST 支持用户上传 PPTX / PDF / DOCX 三类样本，单文件上限 50MB，批量上传上限 20 份 / 批次。
- **FR-007**：系统 MUST 在解析完成后向用户展示文件名、解析状态、解析页数、解析时间，并允许用户删除任一样本。
- **FR-008**：系统 MUST 在解析阶段对原文执行 PII 检测中间件；命中后 MUST 采用"字段级处置"——保留文档并按规则（手机号/身份证号→脱敏星号、邮箱/客户名→占位符）替换命中字段，在解析摘要中列出被处置字段列表，并在"安全事件"页（见 FR-020）落记录。
- **FR-009**：系统 MUST 将"原始文件 / 解析结果 / 嵌入向量"三类数据分离存储，删除任一样本时三类数据同步清除；但由该样本派生的 SlideAsset 在被草稿引用时 MUST 保留为"孤儿素材"（`source_sample_id` 置 NULL），在"我的素材"中以"来源已删除"标识展示，用户可手动决定保留或清理——此行为与 US6 追加说明一致。
- **FR-010**：系统 MUST 对重复样本（哈希相同）执行去重，跳过解析并提示"样本已存在"。

**Agent 进化主线 (P2)**

- **FR-011**：系统 MUST 记录每次生成 / 修改 / 撤销行为，并在用户维度上归纳为"偏好规则"（含来源片段链）。偏好规则的生效条件为：同一"同类型修改"（见"术语定义"）在用户历史中出现 ≥3 次且被忽略次数 <2 次，此时规则状态标记为 `active`。
- **FR-012**：系统 MUST 在新任务中自动应用状态为 `active` 的偏好规则，并在生成轨迹中以"已应用偏好 P-NNN（依据：…）"形式明示。
- **FR-013**：系统 MUST 提供"我的偏好"页面，允许用户查看每条规则的来源片段、应用次数、最近应用时间，并支持手动删除。
- **FR-014**：系统 MUST 在用户对某次自动应用执行"撤销 / 锁定原文"时，将该规则标记为"已忽略一次"但不删除，且不再次自动应用到同一任务。

**可解释可控制 (P2)**

- **FR-015**：系统 MUST 为每次生成产出一份"生成轨迹"，至少包含大纲生成 / 单页要点 / SVG 排版 / PPTX 导出 4 个阶段，每阶段含输入摘要、输出摘要、耗时、引用样本 ID。
- **FR-016**：系统 MUST 允许用户对生成轨迹中任意阶段执行"重做此阶段"，且仅重新执行该阶段及后续阶段。
- **FR-017**：系统 MUST 允许用户在生成中点击"撤销 / 锁定原文"，覆盖当次任务的偏好自动应用，且不污染偏好规则。

**安全与合规 (P3)**

- **FR-018**：系统 MUST 提供"导出我的全部数据"入口，输出包含原始文件、结构化摘要、偏好规则 JSON 的 ZIP 包，处理时间不超过 10 分钟。
- **FR-019**：系统 MUST 提供"删除我的全部数据"入口，二次确认后 24 小时内完成生产库与向量库的全部清除，备份在 7 天内清除。
- **FR-020**：系统 MUST 在"安全事件"页面展示 PII 检测中间件的命中记录，含调用时间、命中字段、处置方式。
- **FR-021**：系统 MUST 禁止在 MVP 阶段默认开启"模型微调"或"上传至第三方训练语料"等隐式行为，并在隐私政策中显式声明。
- **FR-038**：系统 MUST 支持主流浏览器（Chrome ≥120、Edge ≥120、Firefox ≥121、Safari ≥17）的基本交互，核心流程（上传样本 → 输入需求 → 下载结果）在上述浏览器中 MUST 功能完整可用；UI 在 1366×768 及以上分辨率下 MUST 无布局错位。
- **FR-039**：系统 MUST 为所有交互元素提供键盘可访问性（Tab 导航、Enter/Space 触发），关键操作（上传、生成、下载、删除）MUST 有 ARIA 标签，色彩对比度 MUST 满足 WCAG 2.1 AA 标准（≥4.5:1 正文 / ≥3:1 大文本）。

**数据生命周期 (横切)**

- **FR-026**：系统 MUST 对样本与偏好规则采用"长期保留"策略，仅在用户主动删除或触发"删除我的全部数据"时清除。
- **FR-027**：系统 MUST 对生成任务结果保留 180 个自然日（自任务创建日起算）；到期前 14 天向用户发出"续期 / 归档 / 导出"三选一通知，逾期未操作则自动归档至冷存储并使原下载链接失效，任务详情页显示"已归档"标识。用户订阅到期或账号休眠期间，保留策略不变（仍为 180 天），不因订阅状态变化而提前清除或延长。
- **FR-028**：系统 MUST 在每次生成完成后计算并返回"风格契合度"客观指标三层（版式结构 / 配色分布 / 字体族）得分，且单层命中 ≥ 80% 视为该层契合；客观指标全层契合时任务标记为"风格契合"。
- **FR-029**：系统 MUST 限制单用户（按 `user_id` 判定，跨设备/会话全局共享）最多 2 个并发生成任务；第 3 个起入队等待并按 FIFO 调度，且 MUST 在 UI 顶部常驻条展示"当前排队位置 / 预计等待时长"；排队超过 5 分钟未执行则自动取消并通知用户。计划性维护窗口期间（FR-025），排队任务正常保留不取消，维护结束后按原队列顺序恢复调度。

**质量门禁 (横切)**

- **FR-022**：系统 MUST 在每次生成任务结束后保留执行 trace，含 `request_id` / `user_id` / `feature` 三标签，保留时长不少于 90 天。
- **FR-023**：系统 MUST 暴露健康检查与"当前任务队列长度"两个最小运维指标。
**可用性 SLA (横切)**

- **FR-024**：系统 MUST 承诺 MVP 阶段（私有化 Beta 期）月度可用率 ≥ 99%（核心链路：生成 / 解析 / 偏好提取）。
- **FR-025**：系统 MUST 在核心故障（生成 / 解析 / 偏好提取链路完全不可用）发生后 4 小时内恢复，按工作日响应；计划性维护 MUST 提前 24 小时公告。计划性维护窗口期间，正在排队的生成任务（FR-029）MUST 正常保留不取消，维护结束后按原队列顺序恢复调度。


### Key Entities

- **用户 (User)**：平台使用者，含个人版 / 团队版 / 企业版三种角色。核心属性：用户 ID、注册时间、当前活跃知识库、偏好规则集、订阅等级。
- **样本 (Sample)**：用户上传并解析入库的原始资料。核心属性：样本 ID、所有者、上传时间、原始文件路径、解析状态、PII 处理摘要、嵌入向量引用。
- **偏好规则 (Preference)**：由 Agent 从用户历史行为中归纳出的可复用规则。核心属性：规则 ID、所有者、来源片段链（可追溯到具体样本 / 修改历史）、应用次数、最近应用时间、被忽略次数、生效状态。
- **生成任务 (Generation Task)**：一次完整的"用户输入 → PPTX"流程。核心属性：任务 ID、所有者、原始需求、知识库快照、生成轨迹（四阶段输出）、最终 PPTX 路径、状态（运行中 / 成功 / 失败 / 取消）、耗时、Token 消耗。
- **生成轨迹阶段 (Trace Stage)**：生成任务中的最小可独立观察单元。核心属性：阶段 ID、所属任务、阶段名、输入摘要、输出摘要、耗时、引用样本 ID 列表、状态。
- **安全事件 (Security Event)**：PII 检测 / 越权访问 / 异常导出等安全相关记录的最小元数据。核心属性：事件 ID、所属用户、事件类型、命中字段、处置方式、发生时间。

## Success Criteria *(mandatory)*

### Measurable Outcomes

**MVP 闭环有效性**

- **SC-001**：在知识库中预置 3 份不同风格样本的前提下，标准测试集（一句话输入，20 个不同主题）的首版 PPTX 在 5 分钟内交付率 ≥ 95%。
- **SC-002**：上述标准测试集中，至少 80% 的生成结果满足"风格契合"判定——风格契合 = 客观指标三层（版式结构 / 配色分布 / 字体族，定义见"术语定义"）自动化命中均 ≥ 80%，且每月 1 次的 5% 抽样盲评平均分 ≥ 4 分（5 分制）。盲评流程：① 从当月成功任务中随机抽取 5%（最少 3 份），② 由 3 名非本项目的产品经理独立评分（1 分=完全不像样本风格，3 分=部分相似，5 分=高度一致），③ 取三人平均分，④ 评分标准文档须在 M1 前由产品负责人签发并纳入测试 fixtures。
- **SC-003**：用户在 3 步以内（输入需求 → 选择样本 → 下载结果）完成单次生成；超出 3 步视为交互失败。

**知识库与隐私**

- **SC-004**：PII 检测中间件对包含手机号 / 邮箱 / 身份证号的样本解析，命中率 ≥ 99%，误报率 ≤ 5%。
- **SC-005**：单样本删除操作执行后 1 小时内，生产库、向量库、对象存储中均不可被恢复检索（此为单样本删除的即时性要求，区别于 FR-019"删除全部数据"的 24 小时全量清除窗口——后者因数据量大需批量处理，故允许更长清除时间）。
- **SC-006**：数据导出 ZIP 包内容完整性自检 100% 通过（含原始文件、摘要、偏好规则三类资产）。
- **SC-013**：生成任务结果在创建后 180 天内可下载；到期前 14 天触达"续期 / 归档 / 导出"通知，且通知触达率 ≥ 95%（按任务最终所有者计）。
- **SC-014**：单用户第 3 个及之后的并发生成任务 MUST 入队等待；排队位置展示延迟 ≤ 1 秒，排队 5 分钟未执行 MUST 自动取消并通知用户。

**Agent 进化与可解释**

- **SC-007**：用户在前 5 次生成中重复 ≥ 3 次的同类型修改（定义见"术语定义"），第 6 次生成时自动应用率 ≥ 70%。
- **SC-008**：自动应用的偏好必须在生成轨迹中以"已应用偏好 P-NNN（依据：…）"明示，明示率 100%。
- **SC-009**：生成轨迹任意阶段的"重做此阶段"操作耗时，相对于完整重做降低 ≥ 60%。

**运营与可观测**

- **SC-010**：核心链路（生成 / 解析 / 偏好提取）任何阶段的失败，都能在 30 秒内被 trace 系统捕获并暴露给运维。
- **SC-011**：MVP 私有化 Beta 客户数量 ≥ 3 家，回收的有效反馈（按"用户故事"格式）数量 ≥ 10 条 / 客户。
- **SC-012**：核心链路月度可用率 ≥ 99%，核心故障从告警到恢复的中位时间（MTTR）≤ 4 小时（工作日口径）。

## Assumptions

- **用户角色**：MVP 阶段以"产品 / 方案人员"为目标用户，团队版 / 企业版的复杂权限（多成员协作、审批流）不在 MVP 范围，推迟到 GA 阶段。
- **技术栈约束**：本 spec 严格遵循 Constitution v1.0.0 §技术栈与架构约束，不再重新选择技术方案；基于 AgentScope 2.0 多智能体编排框架的二次开发技术方案已在项目奠基阶段固化，端到端生成链路（LLM 大纲/要点/SVG 排版/PPTX 渲染）由 AgentScope 2.0 的工具调用与中间件机制统一驱动。
- **网络与设备**：MVP 假设用户在桌面浏览器（Chrome / Edge）环境下使用，分辨率 ≥ 1366×768；移动端 / 桌面客户端封装推迟到 GA。
- **订阅与计费**：MVP 假设采用"配额制 + 订阅升级"模式（个人 29 元 / 月，团队 99 元 / 用户 / 月，企业 299 元 / 用户 / 月 起），具体定价在 M4 里程碑确认。
- **数据合规**：MVP 阶段假设服务部署在国内云上（数据不出境），隐私政策与个保法对齐由法务在 M4 复核。
- **LLM 供应商**：MVP 阶段默认对接国内合规 LLM 提供商（具体厂商在 M1 启动时通过 P0 决策确认），模型回放（replay）机制是测试门禁的硬性前提。
- **样本库来源**：MVP 假设用户自带样本，不预置公共模板库；行业模板作为后续 GA 阶段的能力扩展。
- **跨语言**：MVP 假设以中文为主，英文 / 日文样本"按原文保留"，不强制翻译。
- **数据合规（嵌入向量）**：MVP 阶段使用的 text-embedding-3-small 模型 API 调用涉及向量数据传输至境外供应商服务器。假设：① 传输数据仅为解析后的结构化文本片段（非原始文件），不含 PII（PII 中间件在嵌入前已执行字段级脱敏）；② 法务须在 M1 前确认该传输不构成"个人信息出境"（参照《个人信息保护法》第三十八条）；③ 若法务判定构成出境，须切换为国内合规嵌入模型（如 BGE 系列），相关 FR/SC 不变。
- **LLM 供应商切换**：若 MVP 期间 LLM 供应商发生切换（从供应商 A 切换到 B），FR-004 的 Token 预估基准须在切换后 7 个工作日内基于新供应商的中位数重新校准；SC-001 的 5 分钟交付率基准不变，但允许切换后首个自然周作为"校准期"暂不纳入达标统计。
- **M1–M3 同步推进**：本 spec 描述的是整体 MVP 闭环，但实现时严格按 M1（生成）→ M2（知识库）→ M3（Agent 进化）→ M4（去AI味 + 私有化发布）顺序推进；任何在本 spec 中描述的能力必须明确归口到对应里程碑。
- **M5（新增 1 周切片）— 素材检索与方案拼装闭环**：US6 描述的"老素材 + 新内容"拼装工作流是 PM/方案人员最高频的实操路径，介于 M1（生成）和 M2（知识库）之间；为避免阻塞 M1 交付，US6 的素材检索能力复用 M2 的样本解析（按页切片 + 嵌入），草稿与导出能力以"轻量复用 M1 阶段"形式插桩实现，整段闭环在 M5 切片内串联验收；具体任务拆分见 tasks.md Phase 5。

### User Story 6 - 商机调研后的素材检索与方案拼装 (Priority: P1) 🎯 MVP 核心 (PM/方案人员核心路径)

作为产品 / 方案人员，我刚刚完成一次商机洽谈与甲方需求沟通。我需要快速翻找我过往积累的产品项目 PPT 材料，看其中是否有可直接复用、或可借鉴重组的内容（如某行业的流程图、某竞品的对标分析页、某客户的架构方案页、某项目的 SVG 插图），结合本次甲方的实际需求，与 AI 自动生成 / 我手动编写的内容进行"拼装重组"，形成最终版方案 PPT。

**Why this priority**：
- 这是 PM / 方案人员最高频的真实工作流——商机洽谈后用"老素材 + 新内容"快速拼方案。US1 的"一句话生成"是其中一条 AI 优先的子路径，但 PM 实际工作中**更常**先检索再拼装。
- 直接兑现产品规划报告中"个人知识库是核心差异点"的承诺（§1.1），把"知识库"从"生成用的素材池"升级为"可被 PM 主动检索和调用的资产库"。
- 缺这条线，MVP 闭环对真正做方案的人是"功能齐了但路径别扭"，会显著降低付费转化。

**Independent Test**：
- 准备 5 份历史项目 PPT（覆盖 3 个不同行业、含 60+ 张可独立页），完成入库解析。
- PM 输入"新能源储能项目立项汇报，需要一张储能系统架构图和一张投资回报分析图"。
- 在 30 秒内搜到至少 3 张可复用页（架构 / 数据图 / 流程图各 1），能直接插入当前草稿。
- 与 AI 自动生成的若干页 + 手动写的 2 页一起，拼出最终方案 PPTX，每张页能溯源（"来自哪份历史 PPT 第几页" / "AI 生成" / "手动"）。

**Acceptance Scenarios**：
1. **Given** 用户知识库中已存在 5+ 份历史 PPT、累计 60+ 张可独立页，**When** 用户在素材库输入关键词"储能系统架构图"，**Then** 系统在 1 秒内返回按"相关度"排序的可复用页列表，每条含：原 PPT 文件名 / 第几页 / 缩略图 / 关键文本片段 / 视觉类型（架构图/流程图/数据图/封面/正文/结语）标签。
   **Given** 用户知识库中无任何历史 PPT（素材库为空），**When** 用户打开素材库页面，**Then** 系统展示"素材库为空，请先在知识库中上传样本 PPT"的引导提示，并提供跳转到知识库上传页的链接，不展示空白搜索框。
2. **Given** 用户已选 3 张复用页与 AI 自动生成的 5 张页拼成 8 张草稿，**When** 用户点击"导出最终方案"，**Then** 系统输出一份 PPTX，文件结构与视觉风格在新老页之间一致（风格三维度中任意两维度命中 ≥80% 的页面占比 ≥90%），且每张页保留"来源"标注（来自哪份样本 + 页码 / AI 生成 / 手动）；未达标的页面在导出摘要中列出差异维度，用户可选择"接受当前样式"或"对该页执行风格归一"后再导出。
3. **Given** 用户选中一张来自风格 A 样本的页，**When** 用户将其插入到一份整体风格 B 的草稿中，**Then** 系统 MUST 提供"风格归一"开关：开启时尝试把该页的配色 / 字体 / 版式参数调整为草稿整体风格；关闭时保留原样并提示"该页与整体风格存在差异"。
4. **Given** 用户在草稿中插入了 3 张复用页、2 张 AI 页、1 张手动页，**When** 用户对某张复用页做修改并保存，**Then** 系统 MUST 不污染原历史样本（解耦），且复用页可在"我的素材"中保留"可再次复用"标记。
5. **Given** 用户在素材库中浏览时，**When** 用户点击"按视觉类型筛选 / 按行业标签筛选 / 按来源项目筛选"，**Then** 系统在 1 秒内更新结果，且筛选状态可在 URL 中分享与回放（团队协作时可用）。

## Clarifications 追加

### Session 2026-06-24 (US6 增补)

- Q: US6 中"复用页"的存储模型——直接引用原样本的二进制 vs 抽离为独立素材资产 → A: **抽离为独立 SlideAsset**。解析阶段在 sample 之外再生成一份"按页切片的素材资产"（保留 SVG / 文本 / 缩略图 + 视觉类型标签 + 嵌入向量），删除原 sample 时，引用过该 sample 的 SlideAsset 仍可保留为"孤儿素材"由用户决定去留；这一决定让素材可独立编辑、可独立二次检索、可在用户跨设备同步。
- Q: "风格归一"开关的默认行为 → A: **默认开启，但提供"原样保留"二级开关**。开启时系统用目标草稿的整体风格（用占多数的页风格聚类）去调整复用页的色板 / 字体族 / 版式参数；归一失败的页（如 SVG 含无法参数化的渐变）保留原样并打"未归一"标签，用户可手动微调。
- Q: 草稿与"一次生成任务"的关系 → A: **草稿是一等公民，生成任务是草稿的一种来源**。一份草稿可由 0+ 个生成任务的输出页 + 0+ 个复用页 + 0+ 个手动页拼成；草稿可"另存为新样本"反向回流到知识库（写入 user-tier 标识，避免污染主知识库）。
- Q: 素材检索的"相关度"评分口径 → A: **文本相似度（BM25）0.4 + 视觉类型匹配（标签命中）0.2 + 嵌入向量余弦 0.4 加权求和**；同分按"被复用次数"降序；用户可对单条结果点"不相关"以降低后续评分。
- Q: 草稿的并发与自动保存 → A: **客户端每 5 秒 debounce 自动保存到服务端；服务端 1 秒内返回保存结果**；草稿同一时间仅允许一个用户编辑（无冲突合并），并发打开时后者只读。

## Requirements 追加 (US6)

- **FR-030**：系统 MUST 在样本解析阶段按"页"为最小单位抽取素材，输出包含 SVG / 文本 / 缩略图 / 视觉类型（cover / toc / architecture / flowchart / data / body / closing）/ 嵌入向量的结构化记录，并入库为独立 `SlideAsset` 资源。
- **FR-031**：系统 MUST 暴露"素材库"页，支持全文检索（BM25）+ 语义检索（嵌入向量）双路召回，按相关度排序返回可复用页列表，单次响应延迟 ≤ 1 秒。
- **FR-032**：系统 MUST 支持按视觉类型 / 行业标签 / 来源项目三个维度筛选素材；筛选状态 MUST 序列化为 URL query string（格式：`?visual_types=cover,flowchart&industry_tags=新能源&source_sample_ids=xxx`），该 URL 格式在筛选维度扩展（新增维度）时 MUST 向后兼容——旧 URL 中未包含的新维度按默认值（全选）处理。
- **FR-033**：系统 MUST 暴露"草稿"一等资源，允许用户在草稿中混合三种来源的页（复用 / AI 生成 / 手动），并对每张页保留来源标注（"来自样本 X 第 Y 页" / "AI 生成于任务 T 第 Z 阶段" / "手动"）。
- **FR-034**：系统 MUST 在插入复用页时支持"风格归一"开关；默认开启，将复用页的色板 / 字体族 / 版式参数向草稿整体风格（见"术语定义"）靠拢；批量插入时逐页独立执行归一，成功页直接应用，失败页保留原样并打"未归一"标签，最终以汇总结果通知用户（如"8 页归一成功，2 页保留原样"）。归一操作仅作用于 SlideAsset 的派生副本（`materialized=true`），不修改原素材文件，因此与 FR-037 无冲突。
- **FR-035**：系统 MUST 对草稿提供自动保存（客户端 5 秒 debounce，服务端 1 秒内确认）与单写者并发控制（多端打开时后者只读）。
- **FR-036**：系统 MUST 在导出草稿为 PPTX 时，输出文件保留每张页的来源标注作为元数据（XMP / 自定义 XML 节点），便于后续溯源与审计。导出过程中若发生异常（服务端错误 / 存储空间不足 / 源文件缺失），系统 MUST 在 30 秒内向用户返回失败原因（RFC 7807 格式），保留已生成的部分文件供用户重新尝试，并记录导出失败事件到安全事件日志。
- **FR-037**：系统 MUST 不污染原历史样本：插入 / 修改草稿中的复用页时，所有变更（包括风格归一后的色板/字体/版式调整）均作用于 SlideAsset 的派生副本（`materialized=true`），原 sample 与其切片不被动。风格归一（FR-034）读取原样本参数用于计算目标值，但仅将结果写入派生副本，不回写原素材。

## Key Entities 追加 (US6)

- **可复用素材 (SlideAsset)**：从样本 PPT 抽取的、按页为单位的独立素材资产。核心属性：素材 ID、所属样本 ID、源页码、视觉类型（cover / toc / architecture / flowchart / data / body / closing）、内容类型（svg / text / image / chart）、缩略图 URL、SVG 原文 / 结构化文本、嵌入向量、行业标签、最近复用时间、复用次数、materialized 标志。
- **草稿 (Draft)**：用户当前正在编辑、可混合多种来源的"工作 PPT"。核心属性：草稿 ID、所有者、标题、整体风格（配色 + 字体族 + 母版）、最后保存时间、当前编辑用户 ID（用于单写者锁）、状态（active / archived / exported）。
- **草稿页 (DraftSlide)**：草稿中按顺序排列的页。核心属性：草稿页 ID、所属草稿 ID、顺序号、来源类型（reused / generated / manual）、来源引用（sample_asset_id / generation_stage_id / null）、派生副本（materialized JSON + 缩略图）、样式覆盖（用户手动调整的色板 / 字体 / 版式）、来源标注文本。

## Success Criteria 追加 (US6)

- **SC-015**：在素材库中，典型查询（10 词以内、含 ≥ 1 个视觉类型关键词） MUST 在 1 秒内返回至少 3 条相关度 ≥ 0.6 的可复用页（相关度 = BM25×0.4 + 视觉类型匹配×0.2 + 嵌入向量余弦×0.4，见 Clarification 定义）；用户后续 5 次同类查询的"首条点开率"（"点开"= 用户点击缩略图后进入素材详情面板）≥ 60%。
- **SC-016**：风格归一开启后，插入的复用页与草稿整体风格的"色板 + 字体族"主观一致度 ≥ 4 分（5 分制）。盲评流程：① 每季度从归一结果中随机抽取 10 张页，② 由 10 名非本项目的评测人员独立评分（1 分=明显不协调，3 分=基本协调，5 分=完全一致），③ 取 10 人平均分，④ 评分标准文档须在 M4 前签发。归一失败的页 MUST 打"未归一"标签且失败率 ≤ 10%。
- **SC-017**：草稿自动保存 5 秒 debounce 触发后，1 秒内返回服务端确认；同一草稿被两端同时打开时，后者进入只读态且 MUST 显示"X 正在编辑"提示。
- **SC-018**：导出草稿为 PPTX 后，使用 XMP / 自定义 XML 节点可还原每张页的来源标注（"来自样本 X 第 Y 页" / "AI 生成于任务 T" / "手动"），溯源成功率 100%。
